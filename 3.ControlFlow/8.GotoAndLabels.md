# Goto and labels

C provides the infinitely-abusable `goto` statement and labels to branch to. Formally, this statement is never necessary and in practice it is almost always easy to write code without it.

Nevertheless, there are a few situations where `goto`s may find a place. The commonest is to abandon processing in some deeply nested structure, such as breaking out of two or more loops at once. 

```c
for ( ... )
    for ( ... ) {
     ...
        if (disaster)
            goto error;
    }
error: 
 /* clean up the mess */
```
A label has the same form as a variable name and is followed by a colon.
