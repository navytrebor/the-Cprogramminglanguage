# Basics of functions

Refer to code in ./FunctionAndProgramStructure/findpattern.c

Each function has the form:

```c
return-type function-name (argument declarations)
{
    declarations and statements
}
```

Various parts may be absent. A minimal function is
```c
dummy ( ) {}
```
which does nothing and returns nothing. A do-nothing function like this is sometimes useful as a place holder during program development. If the return is omitted, `int`is assumed.

A program is just a set of definitions of variables and functions. Communication between the functions is by arguments and values returned by the functions, and through external variables. The functions can occur in any order in the source file, and the source program can be split into multiple files, so long as no function is split.

The return statement is the mechanism for returning a value from the called function to its caller. Any expression can follow return:

```c
return expression;
```

## Compile and Run a Program with Multiple Source Files

When a C program is split across files, each `.c` file is compiled separately, then all object files are linked into one executable.

Example files:

- `main.c`
- `get_line.c`
- `strindex.c`

### Option 1: Compile and Link in One Command

```bash
gcc -std=c11 -Wall -Wextra -pedantic main.c get_line.c strindex.c -o findpattern
```

Run it:

```bash
./findpattern
```

### Option 2: Compile Separately, Then Link

Compile each source file into an object file:

```bash
gcc -std=c11 -Wall -Wextra -pedantic -c main.c
gcc -std=c11 -Wall -Wextra -pedantic -c get_line.c
gcc -std=c11 -Wall -Wextra -pedantic -c strindex.c
```

Link the object files into one executable:

```bash
gcc main.o get_line.o strindex.o -o findpattern
```

Run it:

```bash
./findpattern
```

### Notes

- Function declarations (prototypes) used by multiple files should be placed in a header file, for example `findpattern.h`, and included with `#include "findpattern.h"`.
- `main.c` should contain `main`, while helper functions like `get_line` and `strindex` are implemented in their own `.c` files.
- "Load" in this workflow means executing the compiled program (`./findpattern`) after build and link complete.

## Addendum: Meaning of Common gcc Flags

In the commands above, these options improve portability and code quality:

- `-std=c11`: Compile using the C11 language standard. This gives predictable language rules and avoids compiler-specific defaults.
- `-Wall`: Enable a broad set of common warnings (for example, suspicious constructs and likely mistakes).
- `-Wextra`: Enable additional warnings that are not included in `-Wall`.
- `-pedantic`: Warn about non-standard code that does not strictly follow the selected C standard.

Tip: write `-Wextra` with the leading dash. `Wextra` (without `-`) is treated as a filename, not as a compiler option.
